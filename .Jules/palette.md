## 2024-05-07 - Interactive Elements as Semantic Buttons
**Learning:** In React applications, using `<div>` or `<span>` with `onClick` handlers creates accessibility barriers, as they lack semantic meaning, implicit ARIA roles, and default keyboard navigation (Tab-focusability and Enter/Space activation). While it's possible to manually add `tabIndex="0"`, `role="button"`, and keyboard event handlers, it's brittle.
**Action:** Always prefer native semantic elements like `<button type="button">` for interactive elements that don't navigate to a new page (which should use `<a>`). Apply appropriate `focus-visible:` classes to explicitly style the keyboard focus state without interfering with mouse click outlines. For icon-only buttons, always ensure an `aria-label` or `title` is present.
